HTML

  • most meta elements are not visible to the user (except title)
  • h1 headers should only be used once per webpage
  • the head element contains information about the webpage
  • the body element represents the visible content shown to the user
  • elements nested under an element (parent) are referred to as a child element

CSS

  • CSS is the code that tells the browser what the content should look like
  • CSS work is referred to as front-end development (UI)
  • the HTML file needs to contain a link in the header to recognize its connection to the CSS file
  • margin indicates how much space we want around the outside of an element.
  • padding indicates how much space we want around the content inside an element.
  • you can assign a class to sections of your HTML body, and then apply CSS rules to those specific classes

Git

  • git status: checks what branch we are currently working on in bash
  • git checkout -b branch-name: creates a new branch and switches to it
  • a repo (repository) is where all of the files for a project go
  • coding and modifications should always happen on a different branch from the main
  • to merge coding to the main branch, a pull request needs to be submitted, approved by the team, and then finally merged into the main branch

JavaScript

  • JavaScript adds interactivity to webpages
  • for JavaScript coding, the order of your lines matters; for example, you need to first define your variable before logging it to the console
  • A value is considered falsy if it is one of the following values: 0, -0, On, "", null, undefined, NaN, or a Boolean data type with the value of false. All other values, including a Boolean with the value of true, are considered truthy. (lesson 3.3.4)
  • a variable is a named container that allows us to store data in our code
  • control flow is the order in which a computer executes code in a script
  • you can randomize which choice in an array is selected using math functions
